jQuery KeyValueEditor

Default behavior

The jQuery KeyValueEditor plugin can be used to create a UI for adding key-value-pairs. There are two modes of input: a form with separate text boxes for individual keys and values, and a textarea to copy-paste larger blocks of text.

The conversion to/from the two modes of input is customizable. The default functions split the text into rows, and the individual rows by ':', to give keys and values (for HTTP headers). Customizable options:

  • deleteButton: The string//html to show to delete a row. Default: "Delete"
  • toggleButton: The string//html to show to switche between the form and textarea views. Default: "Toggle view
  • placeHolderKey: The placeholder for the key textbox. Default: "Key"
  • placeHolderValue: The placeholder for the value textbox. Default: "Value"
  • textToFormFunction: Function to convert text to key-value pairs.
  • formToTextFunction: Function to convert key-value pairs to text.


This is the default usage of the plugin. No options are set.

$("#kve_default").keyvalueeditor('init');

The values can get by calling getValues.

var values=$("#kve_default").keyvalueeditor('getValues');

The values can add programatically by calling addParams

$("#kve_default").keyvalueeditor("addParams",[{key:"foo",value:"bar"}]);

Plugin with custom options.

var params = {
   placeHolderKey:"Key_custom",
   placeHolderValue:"Value_custom",
   toggleButton: "Toggle_custom",
   deleteButton:'<img class="deleteButton" src="img/delete.png">'
};
$("#kve_custom").keyvalueeditor('init',params);
